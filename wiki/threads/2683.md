Everything else I have in a separate .gsc file.<br /> <br /><span style="color:#00ffff;">CREATING A GSC FILE</span><br /> <br />The easiest way to create a .gsc file is to copy an existing one and rename it. Though we can have .gsc files in other directories than (Bo3Root\usermaps\yourmapname\scripts\zm) to keep things simple we&#39;ll stay inside that directory. <br /> <br />Copy any .gsc there, such as yourmapname.gsc, and rename it to anything you like. In this example, I will call it my_external_script :<br /> <br /><img style="max-width: 500px;" src="https://imgur.com/sES2c7D.png"><br /> <br />Next thing we will have to do is parse it in the yourmapname.zone. This will let launcher know that we&#39;d like to use this file at some point when our map is run.<br /> <br />Let&#39;s open our yourmapname.zone (located in Bo3Root\usermaps\yourmapname\zone_source). It should look something like this:<br />{% highlight php %}

 <br />To parse our script, we add the line:<br />{% highlight php %}

 <br />to the bottom.<br /> <br /><span style="color:#00ffff;">CALLING AN EXTERNAL FUNCTION</span><br /> <br />Now we have our script recognized by launcher, however this does not mean it will be run. We still need to call it from a script that will already be run, such yourmapname.gsc.<br /> <br />Before we get to that however, let&#39;s give this script something to do. Delete everything inside the .gsc (you can select all with ctrl + a) so we can start from scratch.<br /> <br />Let&#39;s add another simple print function like in the last tutorial.<br />{% highlight php %}

 <br />Since we also want this function to be called after all players have connected, and then 10 seconds after that, we should call it below the wait(10); line.<br /> <br />the way you call an external function is basically the same as a normal function:<br />{% highlight php %}
my_external_script::my_cool_print_function();{% endhighlight %}
 <br />notice:<br /><ul><li>the name of the .gsc file is first<br /><br /><li>followed by two colons <span style="color:#ffcc00;">::</span><br /><br /><li>followed by the name of the specific function within the .gsc file we want to call.<br /><br /></li></li></li></ul>We can also thread it like we did with normal functions:<br />{% highlight php %}
thread my_external_script::my_cool_print_function();{% endhighlight %}

 <br /> <br /><span style="color:#00ffff;">#USING A SCRIPT<br /> </span><br />This is almost complete, however if we try to link now, launcher will throw an error like this:<br />{% highlight php %}

 ^1SCRIPT ERROR: No generated data for &#39;scripts/zm/zm_yourmapname.gsc&#39;

 <br /> <br />this is because anytime you call a function from an external .gsc file, you <strong>must</strong> include that file in the #using&#39;s at the stop of the script that&#39;s calling it. <br /> <br />A #using looks like this:<br />{% highlight php %}
#using scripts\zm\my_external_script;
{% endhighlight %}
 <br />notice:<br /><ul><li>the <span style="color:#ffcc00;">#using</span><br /><br /><li>the filepath (automatically assumes we are starting from the folder Bo3Root\usermaps\yourmapname <strong>OR</strong> Bo3Root\Share\Raw)<br /><br /><li>a #using does <strong>not</strong> include the <span style="color:#ffcc00;">.gsc</span> at the end of the filename<br /><br /><li>uses a <span style="color:#ffcc00;">;</span><br /><br /></li></li></li></li></ul> <br />Let&#39;s slap that bad boy in the preamble of yourmapname.gsc and we should be good to go!<br /> <br />{% highlight php %}

 <br /> <br />Now in-game you should notice both the print function inside yourmapname.gsc and the print function in the my_external_script.gsc being called, at pretty much the same exact time.<br /> <br /><span style="color:#00ffff;">#NAMESPACE&#39;S</span><br /> <br />You may see in some scripts a #namespace. These basically designates a different name for the script than its actual filename. A #namespace goes in the preamble and looks like this:<br /> <br />{% highlight php %}
#namespace zombie_utility;{% endhighlight %}
 <br />it&#39;s as simple as putting <span style="color:#ffcc00;">#namespace</span> followed by the name you want to give the script.<br /> <br />Note that:<br /><ul><li>You can only use one #namespace per script. (that wouldn&#39;t make sense anyway)<br /><br /><li>the #namespace only changes the name you use when <strong>calling</strong> the external function. When parsing the script in the mapname.zone or when putting the script as a #using, you must still use the actual filename.<br /><br /><li>Treyarch uses these all the time<br /><br /></li></li></li></ul><span style="color:#00ffff;">USING TREYARCH&#39;S CODE</span><br /> <br />There are a lot of great things you can do with the list of <a href="http://aviacreations.com/modme/scriptdocs/">Engine Functions</a>, however now that we know how to use external functions, we have access to so much more.<br /> <br />Treyarch has tons of scripts that we can look through and use the functions of in (Bo3Root\share\raw\scripts). For example, one function that I like to use often for EE&#39;s and such is the spawn_zombie function, located in (Bo3Root\share\raw\scripts\shared\ai\zombie_utility.gsc)<br /> <br />This lets me spawn a zombie at a specific struct, and I can use it from any .gsc like this:<br />{% highlight php %}
zom = zombie_utility::spawn_zombie( spawner, target_name, location, round_number );{% endhighlight %}
 <br /> <br />I just have to remember to have the #using for the script at the top of my script:<br /> <br />{% highlight php %}
#using scripts\shared\ai\zombie_utility;{% endhighlight %}
 <br />and remember to make sure I&#39;m calling the function by whatever #namespace Treyarch gave that script, <strong>not </strong>necessarily the filename (though in this case they are the same)<br /> <br /><a href="https://forum.modme.co/threads/abnormal202-scripting-tutorials-master.2680/">See ALL Tutorials</a></p>
